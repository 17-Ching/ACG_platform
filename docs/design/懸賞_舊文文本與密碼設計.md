# 《懸賞：ID 還在線上》— 舊文文本 & 關卡 3 密碼設計

> 配合主設計文件使用。以下是 `k_r_o_w` 兩年前的貼文全文，以及站內信通行碼那關的完整出題與解答。
> 所有沉重題材（逃家、家庭陰影、那場派對）一律用暗示與留白，不寫細節。恐怖來自「零回覆」的孤獨感。

---

## PART 1｜`k_r_o_w` 的舊文（依時間排序）

這幾篇是全遊戲的情緒地基。共同特徵：**推文極少、語氣越來越淡、求助被略過**。玩家爬完會有種「原來早就沒人接住他」的寒意。

---

### 舊文①（最早貼文｜關卡 2 的目標｜關卡 3 密碼藏在推文）

```
作者  k_r_o_w
標題  [心情] 有人半夜也睡不著的嗎
時間  兩年前 03:47

不想回去。
在超商坐到店員一直看我。
如果有個地方可以不用回家就好了。
反正也沒人發現我不在。

—— 手機發文，排版亂請見諒

推 路人甲：生活好無聊，路過推一下
推 abc123：日出了還在爬文，我也是醉了 zzz
推 小明：就這？沒圖沒真相 XD
推 夜貓：是在哈囉，這板怎麼這麼冷
推 匿名：密的啦，樓主加油（隨手推）
推 過客：碼的，怎麼都沒人正經回你，同情推
```

> **設計註**：這篇同時是關卡 2 的終點（玩家從「回到我開始的地方」找到最早貼文）與關卡 3 的起點（密碼藏在推文，見 PART 2）。內文用「反正也沒人發現我不在」一句側寫失蹤前的孤立，不直述家庭狀況。

### 舊文②（家庭陰影，全靠留白）

```
作者  k_r_o_w
標題  [問卦] 你們家晚上會很安靜嗎
時間  兩年前 01:12

我家一到晚上就很吵，但我更怕安靜下來的時候。
門關起來假裝在睡，其實在數幾點天會亮。
算了，沒事，發廢文而已。

推 KKcat：？？發生什麼事了嗎
推 k_r_o_w：沒事啦 想太多
（本文無其他回覆）
```

> **設計註**：唯一一則關心的回覆，被他自己「沒事啦」擋掉。恐怖點在於：有人差一點就接住他了。家暴陰影完全靠「更怕安靜」「數幾點天亮」暗示，不寫任何具體事件。

### 舊文③（生日，沒人記得——與密碼意義呼應）

```
作者  k_r_o_w
標題  [閒聊] 今天生日
時間  兩年前

自己買了顆蛋糕。便利商店那種小小的。
本來想說會不會有人記得，結果手機一整天很安靜。
沒關係，我幫自己唱了。

（本文 0 推 0 噓 0 留言）
```

> **設計註**：`0 推 0 噓 0 留言` 是整篇最恐怖的一行。這篇也暗示了通行碼的意義——「生日就是密碼」（見 PART 2）。玩家解出密碼後回頭看這篇，會更難受。

### 舊文④（那場派對的邀約｜foreshadow，不寫細節）

```
作者  k_r_o_w
標題  [心情] 認識了一個網友 好像終於有人要理我了
時間  失蹤前不久

網路上認識一個人，人超好，一直找我聊天。
他說週末有個很屌的趴，是某個「有錢有勢的大少爺」辦的，
問我要不要去，說去了就懂什麼叫人生。
我猶豫了一下……但好久沒人這樣約我了。
有點期待。

推 KKcat：陌生網友的邀約要小心欸
推 KKcat：真的假的 地址在哪 我不太放心
（樓主未再回覆此篇）
```

> **設計註**：全篇最令人揪心——他難得的「期待」，讀者卻知道結局。派對只寫到「很屌的趴」「大少爺辦的」為止，**不寫藥物、不寫過程**。KKcat 的擔心沒被回覆，接續到站內信那條戛然而止的對話（關卡 4）。

## PART 2｜關卡 3：站內信通行碼（完整出題與解答）

### 謎題機制：藏頭 + 個資交叉

**第一步——發現藏頭。**
舊文①的六則推文看似無意義的路過灌水，但取**每則推文的第一個字**：

| 推文 | 首字 |
|---|---|
| 生活好無聊… | 生 |
| 日出了還在爬文… | 日 |
| 就這？沒圖沒真相… | 就 |
| 是在哈囉… | 是 |
| 密的啦… | 密 |
| 碼的，怎麼都沒人… | 碼 |

直讀 → **「生日就是密碼」**

**第二步——找生日。**
提示指向「生日」。玩家回到 `k_r_o_w` 的**個人資料頁**（頁面結構第 3 項），生日欄寫著例如 `3 / 14`。

**第三步——輸入通行碼。**
站內信匣通行碼 = 生日數字化：**`0314`**。輸入正確 → 解鎖關卡 4 的私訊對話。

### 難度定位
中等。玩家要先意識到「一堆廢話推文是刻意排列的」（藏頭），再把提示與個資頁交叉。不需要外部工具，純站內資訊即可解，但要動點腦。

### 雙關設計（服務真假難辨）
這排藏頭是**被人刻意安排**的——
- 讀成「鬼」：他死後想讓人找到線索，用只有細心人看得懂的方式留話。
- 讀成「局」：有人事先佈置好整條路徑，引導網友往指定方向走。
兩種都成立，遊戲不解答。

### 給創作者的變體選項
- 想更難：把「生日」再加一層——個資頁生日故意被改成假的，真生日藏在舊文③的蛋糕發票圖片 EXIF 裡。
- 想更有年代感：把藏頭改成**注音鍵盤位移**（每則推文末字按注音鍵盤往右一鍵），解出注音再拼字。很對復古 BBS 味，但要附鍵盤對照圖給玩家，否則會過難。
- 想更好實作：藏頭維持前端純文字即可，通行碼用前端驗證，錯誤時回一句 in-world 的話（例如「密碼錯誤。……你真的認識我嗎？」）強化毛感。

## 給創作者的提醒

- 以上人物、帳號、事件全為虛構，請勿影射真實個案。
- 沉重段落已刻意留白，實作時請維持這個分寸，不要往具體化的方向補寫。
- 因觸及高風險少年與物質濫用議題，建議片尾附上求助管道連結，給沉重題材一個溫柔落點。
