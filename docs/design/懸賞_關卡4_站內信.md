## 站內信匣：與 `guest_x` 的對話

介面呈現成老論壇的「悄悄話」列表：左側頭像空白，時間戳精確到分。每則來訊下方有**帶讀取日期的 `已讀` 狀態**——這是本關真正的恐怖來源：前段都是當天已讀，後段的讀取日期卻停在兩年後（見第四、五段）。

### 第五段：更後面（服務真假難辨的核心）

派對三天後，`guest_x` 的語氣變了——像是有別人在用這個帳號，或同一個人開始慌了：

```
guest_x    你把我們的對話刪掉 現在!
                                              已讀 2016/08/29
guest_x    這個帳號以後不要再上線。
                                              已讀 2016/08/29
（此後 guest_x 未再出現）
```

然後——**兩年後**，也就是懸賞文出現的同一時期，這串對話最底下，突然多出一則：

```
k_r_o_w    我讀了。
                                              2016/08/29 00:03
```

> **設計註**：這是全關的引爆點，也是「真假難辨」的心臟。
> - **guest_x 那兩則**：「刪掉對話」「別再上線」——強烈暗示有人在善後、在滅口，支持「這是一場被掩蓋的犯罪＋有人在做局」的讀法。
> - **兩年後那則「我讀了」**：停了兩年沒人讀的訊息，讀取日期全部是 2016/08/29——懸賞文出現的那一天；回覆時刻 00:03，與個資頁「最後上線」同一刻（見總表）。是誰讀的？支持「他還在線上」的讀法。
> - 遊戲**不解答**。玩家自己對出這幾個時間戳，會頭皮發麻——然後自己嚇自己。

## 實作小技巧

- 已讀狀態帶讀取日期（灰字）：前段是當日、後段整排停在 2016/08/29，日期落差本身就是壓迫感，不需任何解說文字。
- 最後那則「我讀了」做成**玩家停留在此頁 30 秒後才淡入出現**（JS 計時，秒數見總表第六節），製造「它是趁我在看的時候出現的」錯覺。
- `guest_x` 全程不給真實身分，IP／註冊資料留白，維持「可能是兇手陣營、也可能只是個中間人」的模糊。
