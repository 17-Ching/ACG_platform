# 《懸賞：ID 還在線上》— 解謎遊戲設計文件

> 類型：偽裝復古匿名論壇的 ARG／心理驚悚（真假難辨）
> 平台：架空的懷舊 BBS／論壇（不綁真實平台，年代感自訂）
> 難度：中等（交叉比對推文、爬帳號歷史、輕度解碼、人肉搜索式拼圖）
> 核心體驗：一個沒人在乎的孩子死後在論壇發起懸賞尋寶，網友越玩越發現不對勁——到底是有人在操弄這個局，還是死者真的還在線上？

---

## 一、核心設定

一個架空的老論壇（自訂站名，如「夜貓板」）。某天，一個沉寂多年的帳號 **`k_r_o_w`** 突然發文：

> 「我藏了東西。找到的人，[懸賞金]是你的。規則很簡單，跟著線索走就好。」

發文者自稱知道一個「大人物」的秘密，懸賞金高得不合理。網友半信半疑跳坑，一關一關往下解。

但越查越冷：這個帳號的註冊資料指向一個**兩年前就失蹤的國中生**；他的舊文透著逃家、家庭失能的痕跡；而他「最後上線時間」，是在他被通報失蹤**之後**。有人開始人肉這個 ID，卻挖出更不對勁的東西——

最後所有線索指向：他死在一場派對，權貴子弟的派對，用藥過量。而那位政治人物父親，為了保住兒子和自己的名聲，把這個「反正沒人會找的孩子」處理掉了。

玩家在終局面臨選擇：**把證據公諸於世，或收下那筆突然匯進來的「封口費」。**

## 二、真假難辨的雙解釋（設計靈魂）

「刻意曖昧」意味著每條線索都要能同時支持這兩種讀法，遊戲**永遠不明說**是哪一種：

| 讀法 | 玩家會怎麼腦補 | 同一條線索怎麼雙關 |
|---|---|---|
| **鬼在發文** | 「他死了還在線上，想有人替他討公道」 | 深夜自動回覆、最後上線時間在失蹤之後、回文語氣像本人 |
| **有人做局** | 「是知情者／記者／甚至兇手陣營在放餌」 | 有人能拿到他的帳號密碼、懸賞金真的匯得出來、線索被精心設計過 |

**寫作鐵則**：任何一條證據，寫完都要自問「這能不能同時證明另一種？」不能，就改到能。全程沒有一槌定音的證據。

## 三、把「沉重題材」處理好的原則

主角是未成年、涉及逃家、家暴陰影、被迫用藥致死。這些**只用暗示與留白**，不寫細節、不寫成過程、不美化：

- 家庭創傷 → 用他舊文的片段、沒回應的求助文、生日沒人祝賀來側寫，不直述。
- 用藥致死 → 只到「一場他不該去的派對」「醒不過來」為止，不描寫藥物、劑量或過程。
- 毀屍滅跡 → 用「找不到、查無此人、紀錄被抹掉」的官僚冷感來呈現，比血腥更毛。

留白在心理驚悚裡本來就更有後勁——恐怖來自「一個孩子消失了，體制運轉如常」的寒意。

## 四、論壇頁面／版面結構

做成靜態網站即可，用版面、文章串、個人資料頁、站內信、404 串起來。

1. **看板列表（首頁）** — 各分版。懸賞文置頂在「都市傳說板」或「協尋板」。
2. **懸賞主文串** — `k_r_o_w` 的發文＋網友回文。遊戲主軸都在這裡推進。
3. **帳號個人資料頁** — 註冊日、最後上線、發文歷史、簽名檔（藏第一層違和）。
4. **舊文考古** — 他兩年前的貼文（側寫背景、埋線索）。
5. **站內信 / 悄悄話** — 需要解出密碼才能看的私訊匣。
6. **相簿 / 附檔區** — 圖片線索（調亮、EXIF、檔名密碼）。
7. **他板的相關討論** — 八卦板有人在聊某政治人物的兒子；協尋板有一則沒人回的尋人啟事。
8. **管理員公告 / 站務** — 揭露帳號被異常登入、IP 對不上。
9. **隱藏頁 / 404** — 終局證據包與結局分歧點。

## 五、線索鏈（中等難度，附解法）

大致線性，中段開分支，讓玩家有「自己在辦案」的手感。

**關卡 1｜違和的帳號資料 → 起疑**
點進 `k_r_o_w` 個資頁：註冊在多年前，但「最後上線」時間晚於他（後面才會查到的）失蹤日。簽名檔是一句沒頭沒尾的話。
- *解法*：純觀察，教學關。先種下「這人是誰」的鉤子。

**關卡 2｜懸賞文的第一道題**
主文要玩家「回到我開始的地方」。→ 指向他的**第一篇舊文**（考古版）。
- *解法*：爬發文歷史找到最早那篇。那篇文透著逃家背景，留言區冷清得刺眼。

**關卡 3｜舊文裡的座標密碼**
最早那篇文的推文有人（另一個帳號）貼了一串看似亂碼的推文，其實是簡單替換／注音鍵盤位移之類的密碼，解開是一組**站內信收件夾的通行碼**。
- *解法*：中等——要玩家意識到「推文也是線索」並解碼。這是第一個真正動腦的關。

**關卡 4｜站內信裡的半截對話**
解鎖的私訊匣裡，有他和一個陌生帳號的對話：對方邀他去一場「很好玩的派對」，語氣熱絡、給地址。對話在某天**戛然而止**，之後全是他單方面已讀不回的訊息。
- *效果*：不寫發生什麼，只讓對話「停在那裡」。玩家自己補完的最毛。

**關卡 5｜相簿的一張圖（分支起點）**
私訊裡有張派對現場的模糊照，檔名是密碼。調亮後隱約看得到背景一個側臉——與八卦板正在被討論的「某政治人物的兒子」對得上。
- *解法*：調亮 / 看 EXIF（拍攝時間、地點與官方說法矛盾）。解出檔名密碼可進下一層。

**關卡 6｜人肉搜索的反轉（雙解釋高潮）**
熱心網友把 `k_r_o_w` 的 IP、註冊信箱、舊照全串起來，做成一份「人肉包」貼在串裡。結論指向那個失蹤國中生。但同一份資料也顯示：**這個帳號近期的登入 IP 換過、有人改過密碼。**
- *效果*：兩種讀法在此正面對撞——是他的鬼？還是有人拿到帳號在做局？遊戲不解答。

**關卡 7｜被抹掉的紀錄**
順著證據去查官方協尋紀錄，卻發現他的案件檔案「查無資料／已結案」，時間點正好在派對之後。體制的冷感成為最大的控訴。
- *解法*：交叉比對日期，拼出「誰有能力讓一個人從系統裡消失」。答案指向那位政治人物父親。

## 六、終局：證據包 + 兩種結局 + 曖昧尾

隱藏頁匯集所有線索，玩家湊齊一份可以扳倒大人物的證據包。然後——一筆錢突然匯進遊戲內的帳戶，附一則訊息：「刪掉你手上的東西，這是你的。」

玩家選擇：

- **結局 A｜揭發**：把證據公開。新聞跑馬燈開始滾動，但結果**留白**——正義是否真的到來？畫面最後停在 `k_r_o_w` 的帳號狀態變成「離線」，像終於能走了。（也可能只是有人登出——曖昧到底。）
- **結局 B｜收下封口費**：畫面安靜，錢到帳，一切歸於平常。但過一陣子，論壇上出現**另一個沉寂帳號的新懸賞文**，開頭一模一樣。循環暗示：局還在轉，你成了共犯。

**共同的曖昧尾**：無論哪個結局，最後都不確認「他到底是不是鬼」。玩家離線後還在想這件事，就成了。

## 七、維持張力的手法（心理驚悚版）

1. **每條證據雙關**：見上表。這是本作最重要的紀律。
2. **介面本身不可靠**：帖子重整後樓層數微變、時間戳矛盾、偶爾「文章已被刪除」再刷新又回來。
3. **冷感勝於血腥**：官僚式的「查無此人」比任何屍體描寫更讓人發涼。
4. **孤獨是主旋律**：他每一篇舊文的零回覆、被略過的求助，才是真正的恐怖來源。

## 八、製作與注意事項

- **技術**：純靜態網站（HTML/CSS/JS）就能做。隱藏頁用不公開路由；「深夜自動回覆」用 JS 讀系統時間；圖片線索用 CSS filter 或另存調亮版；密碼關做前端驗證即可。
- **全部虛構**：站名、帳號、人物、政黨一律原創，**不影射真實政治人物、真實命案或真實失蹤兒少案件**——尊重當事人，也保護你自己。
- **觸及敏感題材的分寸**：逃家、家庭創傷、用藥致死全部用暗示處理，不寫成任何可被當作操作指南的細節。
- **放一個出口**：角落標註「本作為虛構解謎創作」，仿真度越高越需要，避免有人當真報案或人肉。
- **可加關懷資訊**：因為觸及高風險少年與物質濫用，可在片尾放上求助管道連結（如台灣的相關少年／心理諮詢專線），讓沉重題材有個溫柔的落點。
